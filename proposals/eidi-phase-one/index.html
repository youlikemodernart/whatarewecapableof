Phase One for Eidi: lease abstracts.

A focused AI operations build that gives Eidi a useful first tool while keeping Yardi, payments, tenant records, and production systems protected.

Summary

After the second call

The first call was about what an AI-enabled operating layer could do for a firm like Eidi. For the second call, Eidi came prepared with concrete categories of work for the family office. Six came up, each one a workflow the team runs today. This page puts them in an order and recommends where to start.

The first build should be lease abstracts.

Lease abstracts are useful, self-contained, and easy to check. Each field in an abstract can cite the page and clause it came from, so the team can verify the output against the lease itself. The team already reaches for AI when lease questions come up, so the first build extends a habit that exists. It also stays clear of the systems that need the most care.

The other categories still matter. The largest prize is rent payment intake and Yardi posting, and it is also the most sensitive: it touches money, tenant records, and the accounting system at once. Starting with lease abstracts gives both teams a real workflow to judge before that work comes into scope.

Category map

The sequence below separates the work that is ready to prototype now from the work that needs tighter access, process, and security planning.

First build

Lease abstracts

Turn leases into structured summaries of landlord-critical terms, with each field linked back to the source page and clause.

Readiness: ready now. Sensitivity: moderate. Position: recommended Phase One.

Near next

Underwriting intake

Read offering memorandums and portfolio packets against a consistent first-screen view: price per square foot, orientation, traffic, co-tenancy, demographics, and open diligence questions.

Readiness: ready after Phase One or in parallel if needed. Sensitivity: lower, but every source packet still needs approval. Position: strong second build candidate.

Acquisition

Deal and portfolio sourcing

Create a monitored, organized view of opportunities from the approved broker, listing, and special-servicer sources that matter to Eidi.

Readiness: ready with a source list. Sensitivity: medium. Position: good after source approval.

Accounting

Reconciliation support

Make annual CAM, tax, and insurance reconciliations faster and easier to check while keeping Yardi as the accounting backbone.

Readiness: later. Sensitivity: higher. Position: second wave.

Market

Outreach and brand

Sharpen outreach to sellers, brokers, family offices, and special servicers while keeping execution with the team already handling the website, flyers, photos, and email.

Readiness: can run alongside operations work. Sensitivity: lower, but public changes still need approval. Position: parallel lane, with Mike Cartell in the loop.

Later prize

Rent payment intake and Yardi posting

Help with payments coming in, posting to Yardi, and reconciling against leases. This is high-value work, and it is the category that needs the most care.

Readiness: its own phase. Sensitivity: highest. Position: later, after on-site process mapping.

Guardrails

Three commitments keep Phase One moving without pulling sensitive systems into the prototype.

Yardi

Yardi stays the system of record.

We build around it and enhance the work next to it. Nothing we propose replaces or copies it. Any later Yardi work starts read-only and moves toward write access only with a specific reason and an agreed test path.

Data

Phase One runs on samples.

Redacted or representative lease examples are enough to begin. The first build needs no production exports, tenant records, payment data, or system logins.

Payments

Payment work moves slower.

Rent payment intake and Yardi posting need on-site process mapping, a security review, and change control before any build.

Recommended first build

A working lease abstract prototype on a small set of redacted or representative leases. Phase One is sized to answer one question: does it produce abstracts the team uses and trusts?

Phase One

Lease abstracts

Phase One is a focused lease abstract build. The goal is a working prototype that turns a lease into a concise, checkable abstract the team uses in real work.

What we deliver

Deliverable 1

Abstract template

Defined with the team. Co-tenancy, exclusivities, rent bumps, options, expirations, assignment language, prohibited uses, and notice requirements are examples; the team sets the final list of fields.

Deliverable 2

Working tool

A tool that produces a structured abstract from a lease document.

Deliverable 3

Source links

Each extracted field links back to the page and clause behind the answer.

Deliverable 4

Review flags

Terms that need human judgment are marked instead of forced into an answer.

Deliverable 5

Reliability readout

A short readout on reliability, gaps, and what it takes to run the workflow across more of the portfolio.

What success looks like

At the end of Phase One, the team answers one question: does this produce an abstract you use and trust?

If yes, we choose the next build together. If no, we have spent a small, bounded amount of time and we know where the constraints are.

Timeline and terms

A Phase One build like this can run in a few weeks. We can keep commercial terms in the cover note or finalize them once Eidi approves the lease abstract build.

Next

How we want to work

The useful version of this work is a partner close enough to the business to learn the systems, the people, and how Eidi makes decisions. Most of what slows AI work down is access and context. The model is rarely the bottleneck. Phase One is a low-commitment way to start that relationship on work neither team has to worry about.

Continuation test

Phase One should earn the next one. At the end, we review the prototype against real leases and decide together whether it saves enough time and produces output the team trusts.

If it does, we choose the next build: broader lease coverage, underwriting intake, deal and portfolio sourcing, or the deeper payment and Yardi discovery. If it does not, we adjust or stop, with limited time spent.
