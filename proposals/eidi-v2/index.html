A first AI operations build for Eidi.

Start with lease abstracts: a useful, checkable workflow that proves value before higher-sensitivity Yardi, payment, and accounting systems come into scope.

Summary

Recommendation

The second call made the map clearer. Eidi has six useful AI operations categories: lease abstracts, underwriting intake, deal and portfolio sourcing, CAM, tax, and insurance reconciliation, outreach and brand, and rent payment intake with Yardi posting.

The first build should be the lease abstract category.

That category is useful, low-risk, and easy to check. The documents are self-contained. The output can cite the page and clause behind each answer. The team already reaches for AI when lease questions come up, so the first build extends a behavior that exists.

The other categories still matter. Payment intake and Yardi posting may be the largest prize, but that category touches money, tenant records, and the accounting system. Lease abstracts give both teams a real workflow to judge before higher-sensitivity systems come into scope.

Category map

The sequence below separates the work that is ready to prototype now from the work that needs tighter access, process, and security planning.

First build

Lease abstracts

Turn leases into structured summaries of landlord-critical terms, with each field linked back to the source page and clause.

Readiness: ready now. Sensitivity: moderate. Position: recommended Phase One.

Near next

Underwriting intake

Read offering memorandums and portfolio packets against a consistent first-screen view: price per square foot, orientation, traffic, co-tenancy, demographics, and open diligence questions.

Readiness: ready after Phase One or in parallel if needed. Sensitivity: lower, but every source packet still needs approval. Position: strong second build candidate.

Acquisition

Deal and portfolio sourcing

Create a monitored, organized view of opportunities from the approved broker, listing, and special-servicer sources that matter to Eidi.

Readiness: ready with a source list. Sensitivity: medium. Position: good after source approval.

Accounting

Reconciliation support

Make annual CAM, tax, and insurance reconciliations faster and easier to check while keeping Yardi as the accounting backbone.

Readiness: later. Sensitivity: higher. Position: second wave.

Market

Outreach and brand

Sharpen outreach to sellers, brokers, family offices, and special servicers while keeping execution connected to the team already handling the website, flyers, photos, and email.

Readiness: can run alongside operations work. Sensitivity: lower, but public changes still need approval. Position: parallel lane with Mike Cartell in the loop.

Later prize

Rent payment intake and Yardi posting

Help with payments coming in, posting to Yardi, and reconciling against leases. This is high-value work, and it is the category that needs the most care.

Readiness: its own phase. Sensitivity: highest. Position: later, after on-site process mapping.

Guardrails for the category map

Yardi

Yardi stays the system of record.

We build around it. Any later Yardi work starts read-only and only moves toward write access if there is a specific reason and an agreed test path.

Data

Phase One starts on samples.

Redacted or representative lease examples are enough to begin. The first build does not need production exports, tenant records, payment data, or system logins.

Sources

External sources wait.

Broker sites, listing platforms, and special-servicer pages each need terms and access review before automation.

Payments

Payment work moves slower.

Rent payment intake and Yardi posting need in-person process mapping, security review, and change control before any prototype.

Marketing

Public changes stay coordinated.

Website, flyer, photo, and email-blast execution should keep running through Mike Cartell and the existing team. Our role is strategy, direction, and AI-enabled targeting.

Phase One

Lease abstracts

Phase One is a focused lease abstract build. The goal is a working prototype that turns a lease into a concise, checkable abstract the team uses in real work.

What we deliver

Deliverable 1

Abstract template

Defined with the team: co-tenancy, exclusivities, rent bumps, options, expirations, assignment language, prohibited uses, notice requirements, and any Eidi-specific fields that matter.

Deliverable 2

Working tool

A tool that produces a structured abstract from a lease document.

Deliverable 3

Source links

Each extracted field links back to the source page and lease clause.

Deliverable 4

Review flags

Terms that need human judgment are marked instead of forced into an answer.

Deliverable 5

Reliability readout

A short readout on reliability, gaps, and what it takes to run the workflow across more of the portfolio.

What success looks like

At the end of Phase One, the team should be able to answer one question: does this produce an abstract we use and trust?

If yes, we can expand the lease abstract workflow, connect it to underwriting, or open discovery for the payment workflow. If no, we will know where the constraints are after a bounded build.

Timeline and terms

A Phase One build like this can run in a few weeks. We can keep commercial terms in the cover note or finalize them once Eidi approves the lease abstract build.
