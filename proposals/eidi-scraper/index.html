Eidi Deal Watch.

A phased build for watching approved deal sources, catching new or changed opportunities, and turning them into a short digest the Eidi team can review.

Direction

Start with the places Eidi already looks

The next system should start with the places Eidi already checks for opportunities. If ChatGPT is already useful for ad hoc lease questions, Deal Watch is a better first build: a light operating layer around the websites and pages Eidi already uses.

Deal Watch checks approved sources on a schedule, looks for new or changed opportunities, sets aside obvious noise, and puts the best candidates into a digest/review page. Eidi still decides what to pursue. The system narrows the review list and keeps the source work consistent.

The earlier lease-abstract packet should stay paused while this direction is reviewed.

Proposal

Two phases: analyze the sources, then build the watchers

The 20-source list is enough to start, but the sources will not all behave the same. Some will be simple public listing pages. Some may be protected, unstable, login-gated, or better left as manual checks. The analysis phase identifies the best first candidates before the build phase is confirmed.

The build phase is priced as a range because the work depends on what the analysis shows: how each site is structured, what fields are visible, how often it changes, and whether it can be watched cleanly.

Phase 1

Analysis Phase

$3,400 up front | 5 to 7 business days

The analysis phase reviews the initial source list, sets up a few low-friction watchers, and gives Eidi a first digest/review output to evaluate.

Deliverables: analysis of all 20 sources, 2 to 4 low-friction public source watchers, first sample digest/review page, and a recommended build plan.

Phase 2

Build Phase

$7,800 to $9,400 | half up front, half upon completion | 14 to 20 business days

The build phase expands Deal Watch to the approved priority sources, adds a small discovery lane, and iterates the digest/review pages from Eidi feedback.

Deliverables: expanded watched sources from the initial list, source discovery plus 1 to 2 discovery-based watchers where feasible, digest/review-page iterations, source health notes, and a maintenance plan.

The build range will be confirmed after the analysis phase. If the sources are cleaner than expected, the build can include more of the initial list. If several require special handling, they can be separated into manual watch, later expansion, or source-specific approval.

What Deal Watch does

The system should stay practical. It should reduce repeat checking, keep source handling clean, and give the team a better starting point for review.

Input

Approved source list

Eidi provides the listing sites, broker pages, special-servicer pages, relationship pages, and other sources the team already watches.

The first version begins with sources the team recognizes. New-source discovery can follow once the first watch list is working.

Rules

Source-by-source handling

Before a source is watched, it is classified by access type, visible fields, source risk, and recommended cadence.

Public sources, login-based sources, paid sources, and relationship-sensitive sources need different handling.

Watch

New and changed opportunities

Deal Watch checks the approved sources and looks for new listings, changed listings, repeated opportunities, and source issues.

The output should show what changed, where it came from, and why it may deserve a closer look.

Review

Digest and review page

The digest shows new opportunities, changed listings, likely-fit items, and items that need a human check.

The first output should be easy to read, easy to forward internally, and easy to mark up with team notes.

Tune

Better with each pass

Eidi marks each item useful, irrelevant, duplicate, stale, or needs more context.

Those notes sharpen the filter and show which sources are worth watching continuously.

What the digest can show

The field list should follow how Eidi buys. A website may show more, but the digest should pull forward the facts that help the team decide whether to review the opportunity.

FieldUse
Source and linkShows where the opportunity came from and gives the team a path back to the listing.
Property or portfolioNames the asset, portfolio, location, and asset type when listed.
First-screen factsCaptures asking amount if listed, square footage or acreage, occupancy notes, and contact details where available.
Change notesShows when an item first appeared and whether asking amount, status, availability, or source format changed.
Review noteFlags why an item may fit, why it may not, and what information is missing.

What Eidi sends to start

Keep the first ask small. Two inputs are enough to start the analysis phase.

1

Sources to watch first.

URLs are enough: listing platforms, broker pages, special-servicer pages, saved searches, or other sites Eidi already checks.

2

Examples of fit and pass.

If easy, share one or two opportunities worth attention, plus one or two that looked relevant but were passes. Links or screenshots are enough.

With those, the analysis phase can classify the first source set, see how Eidi separates useful opportunities from noise, and start the first digest.

What success looks like

At the end of the first two phases, Eidi should have five practical answers.

1
Which sources are worth watching continuously?
2
Did Deal Watch catch items the team would otherwise check manually?
3
Did the digest save review time?
4
Did the filtering match Eidi's deal judgment?
5
What should be maintained, expanded, or left manual?

If the answers are strong, Deal Watch can become an ongoing service. If the answers are mixed, the next step is to adjust the sources, fields, or cadence before expanding.

Next step

The next step is to approve the analysis phase and share the first source URLs, plus a few fit or pass examples if they are handy. Those are enough to set up the first Deal Watch digest.
