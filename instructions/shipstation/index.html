What are we capable of?
Prepared for Sherri / May 2026

This guide is the working ShipStation process for Ezra Arthur production. It covers the daily queue, Nordstrom tracking rules, label creation, missing order checks, tracking failures, reprints, voids, split shipments, and the package information Sherri should collect.

Summary is the rule and the order paths. Daily is the working queue routine. Ship is how to create and confirm a label. Issues is what to do when an order or tracking goes wrong. Setup is what the shipping station still needs.

Summary Daily Ship Issues Setup ?

Main rule: ship Nordstrom orders through ShipStation. Do not mark Nordstrom orders fulfilled in Shopify.

Nordstrom tracking depends on the order moving through the same chain every time. Nordstrom enters Mirakl, Mirakl syncs into Shopify, Shopify imports into ShipStation, ShipStation creates the label, ShipStation sends tracking back to Shopify, and Shopify plus Mirakl send tracking status to Nordstrom.

If someone marks the Shopify order fulfilled without tracking, Nordstrom may never receive the tracking number. That is the missing tracking problem this process is meant to prevent.

What ShipStation is for

Use ShipStation as the shipping desk for production. Use it to see orders ready to ship, confirm the ship-to address, compare rates, buy and print labels, send tracking back to the selling channel, check marketplace notification status, and reprint or void labels when something goes wrong.

Use Shopify to view order context when needed. Use ShipStation to ship.

Order paths

Channel Working path
Ezra Arthur website Shopify to ShipStation to Shopify. These are normal ezraarthur.com orders. When the label is created in ShipStation, ShipStation updates Shopify with carrier and tracking.
Nordstrom Nordstrom and Mirakl to Shopify to ShipStation to Shopify to Mirakl and Nordstrom. In Shopify, look for tags or notes such as Mirakl, Nordstrom, the Nordstrom order id, and a ship-by deadline.
Nokona WooCommerce to ShipStation is still under validation. Do not change OrderDesk, WooCommerce, or Nokona shipping settings unless Noah asks.

Stop rules

  • Do not click Mark as fulfilled in Shopify for Nordstrom orders.
  • Do not manually create or manually fulfill a Nordstrom order unless Noah gives a specific instruction.
  • Do not use ShipStation Mark as Shipped for Nordstrom unless Noah approves it.
  • Do not ship the same order from two systems.
  • Do not force payment status changes.
  • Do not ignore a failed marketplace notification icon.

Start in ShipStation

Open ShipStation and go to the Orders tab. ShipStation usually displays Awaiting Shipment by default. This is the main working queue for paid orders that are ready to ship.

Step 1

Refresh orders

Click the Import icon in the top toolbar. Choose Update All Stores, or choose the import icon next to the Shopify store if you only need Shopify orders.

ShipStation can auto-import while you are away, but it does not let us set the exact auto-import schedule. If you are about to ship, manually refresh first.

Orders tab with the Import icon and Awaiting Shipment queue visible.

Step 2

Check the queue

In Awaiting Shipment, sort or filter by store, source, order number, requested service, ship date, tags, label created, label printed, and marketplace notified.

For Nordstrom, search by the Shopify order number without the # mark, such as 5011. Cross-check the Shopify order if the deadline or source is unclear.

Step 3

Hold anything unclear

Do not create a label when something looks wrong. Put the order on hold or leave it unshipped and send Noah the issue.

Hold or flag the order when

  • The order is Nordstrom and does not show a clear tracking flow.
  • The address looks incomplete.
  • The package size or weight is uncertain.
  • The order has multiple items and needs split shipping.
  • The product is out of stock.
  • The customer requested something unusual.
  • The order appears fulfilled already but has no tracking.

Ship one order

Step 1

Select the order

In the Orders grid, select the order. The Configure Shipment widget should appear in the shipping sidebar. If the sidebar is hidden, turn on Show Sidebar.

Future screenshot

Selected order with the Configure Shipment widget open in the sidebar.

Step 2

Confirm order details

Before buying the label, check the order number, source, recipient name and address, product SKU and quantity, requested service, ship-from location, package type, dimensions, weight, insurance, signature needs, and ship date.

For Nordstrom orders, also check the ship-by deadline in Shopify tags or notes if it is not visible in ShipStation.

Step 3

Enter packed weight

Use the package's packed weight. If a USB scale is connected through ShipStation Connect, place the package on the scale and click the Scale button in the Configure Shipment widget.

ShipStation supports most HID-compliant USB scales connected directly to the workstation. Avoid USB hubs or extenders for the scale.

Step 4

Choose service and package

Select the carrier service and package type. Enter package dimensions whenever possible. Some services require dimensions, and dimensions help prevent inaccurate rates and carrier fee adjustments.

Current Ezra note: ShipStation imports from Shopify have shown package dimensions as 0 x 0 x 0. Choose the correct package or enter dimensions until presets are created.

Step 5

Compare rates if needed

Use the Rate Browser from the Configure Shipment widget to compare available services. That rate browser uses the order details, address type, package, weight, dimensions, confirmation type, and service class.

The toolbar Rate Browser can create a label without an order. Use that only for one-off shipments that do not need order items, packing slips, or customer notifications.

Step 6

Create and print the label

Use Create + Print Label when the order is ready to ship. ShipStation requests the label from the carrier, creates a shipment record, moves the shipment to Shipped, moves the order to Shipped when every shipment on that order is shipped, and sends shipment information back to the connected store by default.

If you want to create the label now and print later, use the dropdown and choose Create Label.

Print method

Method Use
ShipStation Connect Preferred for the shop. It sends the label to the selected printer and supports Smart Document Tracking.
Download PDF Good fallback. ShipStation still marks the label as printed.
Browser print Usable fallback. Browser print does not mark the label as printed in Smart Document Tracking.

ShipStation Connect works best with USB printers. ShipStation says it does not support Rollo printers through ShipStation Connect. Rollo can still be used through Download PDF or browser print if needed.

Confirm the shipment

  • Go to the Shipments tab or open the shipment record.
  • Confirm the tracking number is present.
  • Confirm the carrier is present.
  • Confirm marketplace notification shows Notified or Sent when the channel should be updated.
  • Check that no red failed notification icon appears.

What tracking should do

ShipStation sends the connected store the shipment tracking number, carrier, order identifier, and notification information. For Shopify, the integration supports sending status update, tracking number, carrier, service, and item information.

This is why the label must be created in ShipStation for Nordstrom orders. ShipStation updates Shopify, then the Shopify and Mirakl connection can update Nordstrom.

Missing from ShipStation

  1. Refresh the store from ShipStation using the Import icon.
  2. Search by order number without punctuation. Example: search 5011, not #5011.
  3. Check Awaiting Shipment, On Hold, Shipped, and Cancelled.
  4. Check Shopify for the order.
  5. For Nordstrom, confirm the Shopify order has Nordstrom or Mirakl tags and is unfulfilled.
  6. If the order still does not appear, stop and tell Noah.

Send Noah the selling channel, Shopify order number, Nordstrom or Mirakl order id if present, what you searched in ShipStation, whether the order is visible in Shopify, Shopify fulfillment status, and any ShipStation error or screenshot.

Do not create a manual ShipStation order for Nordstrom unless Noah tells you to.

Tracking is missing

If the label was created in ShipStation

Check the shipment record

Go to Shipments, search by order number or tracking number, open the shipment record, confirm tracking number and carrier, and check marketplace notification status. If marketplace notification failed, hover over the failure and send Noah the failure reason.

If there is no ShipStation label

Find the label source

Do not add a fake tracking number. Find out where the label was created. Gather carrier, tracking number, ship date, and order number. Send the details to Noah before changing Shopify or Nordstrom.

If it is a Nordstrom order already fulfilled in Shopify with no tracking, tell Noah immediately. That is the failure pattern we are trying to prevent.

Reprint, void, and reship

Action Rule
Reprint Use when the printer failed or the label was lost before it was used. Reprinting prints the same label with the same tracking number and does not buy postage again.
Void Use when the label should not be used. Voiding a label moves the order back to Awaiting Shipment. For Nordstrom, tell Noah before voiding and reshipping if tracking may already have been sent.
Split shipment Use only when one order must ship in more than one package or at different times. The order stays in Awaiting Shipment until every shipment in that order is shipped. For Nordstrom, ask Noah before partial shipment unless approved for that order.
Mark as Shipped Records shipment without buying a label. Use only when a label was created outside ShipStation and Noah approved that path. For Nordstrom, treat this as an exception.

Recommended shop setup

  1. Install ShipStation Connect on the shipping workstation.
  2. Use a USB label printer if possible.
  3. Set label format to match the printer, usually 4 in x 6 in for a thermal label printer.
  4. Set default printers for labels and packing slips.
  5. Connect a HID-compliant USB scale directly to the workstation.
  6. Test the Scale button in the Configure Shipment widget.
  7. Create package presets after Sherri confirms package dimensions and product rules.
  8. Create saved views for common work queues.

Useful saved views

  • Awaiting Shipment, Shopify.
  • Awaiting Shipment, Nordstrom or Mirakl tag.
  • Orders with label created but label not printed.
  • Marketplace notification failed.
  • Orders on hold.
  • Orders with requested service Standard.

Order status reference

Status Meaning
Awaiting Shipment Ready to ship. Main work queue.
Awaiting Payment Do not ship. Labels cannot be created from this status.
On Hold Intentionally paused.
Shipped Label created or order marked shipped. Work is done unless there is a problem.
Cancelled Inactive. Cancel in the selling channel first when possible, then refresh ShipStation.

Label button reference

Button Meaning
Create + Print Label Buy label and print now.
Create Label Buy label now and print later.
Print Label Reprint an existing label.
Void Label Cancel a label that should not be used.
Mark as Shipped Record shipment without creating a label. Use only with approval for marketplace orders.
