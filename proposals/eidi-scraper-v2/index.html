Eidi Deal Watch.

A phased build for turning approved deal sources into one weekly review page: new or changed opportunities, source notes, and the short list most worth Eidi's attention.

Direction

Start with the places Eidi already checks

The next system should start with the places Eidi already checks for opportunities. If ChatGPT is already useful for ad hoc lease questions, Deal Watch is a better first build: a light operating layer around the websites and pages Eidi already uses.

Every week, Eidi gets one review page instead of a pile of tabs. Deal Watch checks the approved sources on a schedule, removes obvious repeats and stale items, and pulls the strongest new or changed opportunities into a short list for the team to review.

If 20 sources take 5 to 10 minutes apiece to check manually, one full pass is roughly 1.5 to 3 hours before follow-up review. The goal is to compress repeat checking into one review surface so Eidi spends time judging opportunities, not reopening the same sources.

The earlier lease-abstract packet should stay paused while this direction is reviewed.

What Eidi gets each week

The operating promise is concrete: approved sources checked on a schedule, one short review page, and a small set of opportunities that deserve attention.

Check

Approved sources reviewed on cadence

Deal Watch checks the source list daily where feasible, or on the cadence that fits each source after analysis.

Protected, login-based, or relationship-sensitive sources stay manual until Eidi approves a specific handling path.

Filter

Obvious noise removed before review

Repeated items, stale listings, low-fit results, and source-format issues are separated from the main list.

The review page should focus attention on new items, changed items, and items that need a human check.

Review

Up to 10 best-fit opportunities

The target weekly output is up to 10 best-fit opportunities from approved sources, plus material listing changes and any source issues worth seeing.

The number can be lower in a quiet week. A useful system should reduce checking work without pretending every week produces 10 good deals.

Proposal

Two phases: prove the output, then build the operating rhythm

The 20-source list is enough to start, but the sources will not all behave the same. Some will be simple public listing pages. Some may be protected, unstable, login-gated, or better left as manual checks. The analysis phase identifies the best first candidates before the build phase is confirmed.

The build phase is priced as a range because the work depends on what the analysis shows: how each site is structured, what fields are visible, how often it changes, and whether it can be watched cleanly.

Phase 1

Analysis Phase

$3,400 up front | 5 to 7 business days

The analysis phase classifies the initial 20 sources, sets up 2 to 4 low-friction public source checks, and produces the first review page for Eidi to evaluate.

Deliverables: all 20 sources classified, 2 to 4 scheduled public source checks, first review page with the strongest new or changed opportunities from those sources, and a recommended build plan.

Phase 2

Build Phase

$7,800 to $9,400 | half up front, half upon completion | 14 to 20 business days

The build phase expands Deal Watch to the approved priority sources, adds daily checks where feasible, and turns the first review page into a repeatable weekly operating rhythm.

Deliverables: expanded watched sources from the initial list, source discovery plus 1 to 2 discovery-based checks where feasible, weekly digest/review-page iterations, target list of up to 10 best-fit opportunities, source health notes, and a maintenance plan.

The build range will be confirmed after the analysis phase. If the sources are cleaner than expected, the build can include more of the initial list. If several require special handling, they can be separated into manual watch, later expansion, or source-specific approval.

What Deal Watch does

The system should stay practical. It should reduce repeat checking, keep source handling clean, and give the team a better starting point for review.

Input

Approved source list

Eidi provides the listing sites, broker pages, special-servicer pages, relationship pages, and other sources the team already checks.

The first version begins with sources the team recognizes. New-source discovery can follow once the first watch list is working.

Rules

Source-by-source handling

Before a source is watched, it is classified by access type, visible fields, source risk, and recommended cadence.

Public sources, login-based sources, paid sources, and relationship-sensitive sources need different handling.

Watch

New and changed opportunities

Deal Watch checks approved sources and looks for new listings, changed listings, repeated opportunities, and source issues.

The output should show what changed, where it came from, and why it may deserve a closer look.

Review

Weekly digest and review page

The digest shows new opportunities, changed listings, likely-fit items, and items that need a human check.

The first output should be easy to read, easy to forward internally, and easy to mark up with team notes.

Tune

Better with each pass

Eidi marks each item useful, irrelevant, duplicate, stale, or needs more context.

Those notes sharpen the filter and show which sources are worth watching continuously.

What the review page can show

The field list should follow how Eidi buys. A website may show more, but the review page should pull forward the facts that help the team decide whether to look closer.

What Eidi sends to start

Keep the first ask small. Two inputs are enough to start the analysis phase.

1

Sources to watch first.

URLs are enough: listing platforms, broker pages, special-servicer pages, saved searches, or other sites Eidi already checks.

2

Examples of fit and pass.

If easy, share one or two opportunities worth attention, plus one or two that looked relevant but were passes. Links or screenshots are enough.

With those, the analysis phase can classify the first source set, see how Eidi separates useful opportunities from noise, and produce the first review page.

What success looks like

By the end of the first two phases, success means observable weekly output: approved sources checked, noise reduced, best-fit opportunities surfaced, and review time compressed.

1
All 20 initial sources classified by access, reliability, cadence, and usefulness.
2
2 to 4 public sources checked in Phase 1, with priority sources added in Phase 2 where feasible.
3
A weekly review page with up to 10 best-fit opportunities, material changes, and human-check items.
4
Manual source checking compressed from roughly 1.5 to 3 hours per full pass into a short review page.
5
Eidi feedback used to decide what should be maintained, expanded, or left manual.

If the answers are strong, Deal Watch can become an ongoing service. If the answers are mixed, the next step is to adjust the sources, fields, or cadence before expanding.
