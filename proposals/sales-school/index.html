The sales school already has the hard part: recorded training, a live delivery rhythm, and Wistia-hosted video content. The current experience wraps that content in an email preview and six PDF indexes. Students can get to the videos. The route feels like a temporary link system.

Build a restrained course portal around the existing content. The first version can stay short of a full LMS. It needs to remove the PDF index layer, give the live session one stable home, and make the videos and resources navigable.

What we found

01

The current path has three layers before a student reaches the actual learning object: email preview, PDF index, Wistia page. That works as distribution. During a timed Zoom break, three layers is a lot of navigation.

02

The curriculum has a natural structure: Day, Breakout, Lesson, Resource. The current interface presents those relationships as buttons and PDF links. A portal can turn the same structure into navigation.

03

The live Zoom context changes the job of the site. Students need to know what to watch now, what order to watch it in, and how to return to the room. A normal video library leaves that live-session question unanswered.

04

Wistia can remain the media layer. The upgrade is the architecture around the videos: current-session access, breakout pages, resource separation, and clearer wayfinding.

The proposed architecture starts with the course as a place, then branches into the live session, the two days, the breakout sections, and the supporting resources.

The strongest interface idea is a live-break console. The course is partly synchronous, so the site should have a current-state screen instead of only a library.

Direction 1: Live Break Console

Primary screen

Today page

A live dashboard shows the active breakout, the watch sequence, and the return instruction. The instructor can share one link during Zoom. Students land on the right thing immediately.

Direction 2: Two-Day Course Map

Course overview

Day One and Day Two as the main structure

The existing two-day structure becomes a map. Each breakout is a card with a theme label, video count, total runtime, and open state.

Direction 3: Lesson Library

Review mode

Findable videos and resources

After the live session, students can return to the library, filter by day or breakout, and find videos or resources without reopening a PDF.

This can be scoped in two layers. The first layer makes the current training feel coherent. The second layer turns it into a fuller learning product.

Phase 1: Course portal

Build

Replace the PDF indexes with pages

Build Course Home, Today, Day One, Day Two, six Breakout pages, Resources, and Facilitator. Keep Wistia as the media layer. Use the existing video URLs and resources.

Output: a navigable training portal that supports live breaks and later review.

Phase 2: Platform layer

Optional

Add search, progress, and admin

Add student progress state, searchable lesson content, transcript indexing, admin editing, cohort management, or login. These are useful if the sales school will run repeatedly.

Output: a reusable learning system instead of a one-off training portal.
